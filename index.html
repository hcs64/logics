A brief explanation of

Formal logic

Formal logic takes its name from dealing with the form of logical statements. It should be possible to identify statements that are logically true, and construct valid arguments, purely at a structural level, once a few basic axioms are granted. If there is an error in the logic, then, it should not have much space in which to hide.

There are two basic approaches to establishing the truth of statements:

  • evaluation (semantically-oriented, this encompasses model checking, satisfiability, etc)
  • proof (syntactically-oriented, theorem proving)

Evaluation consists of taking a well-formed formula (wff) and, based on its structure and rules for the meaning (semantics) of each subexpression, determining the overall meaning of the formula. When the truth of propositions used in the formula are known, this is trivial. When we must instead determine whether there is any assignment of propositional variables that could make the expression true, i.e. whether the statement is satisfiable, this becomes a rather different problem, which is very difficult in general.

Formal proofs are a stylized form of argument, taking the approach of generating new true statements from previously accepted or proven statements. This is done entirely on a syntactic level. It can be used to determine the truth of statements only insofar as any statement for which a proof exists can be considered true. This may be approached by working backwards from the formula to be proved, trying to show how it follows from the axioms. The fundamental mechanism here is modus ponens, if p implies q is true and p is true, then q is true. A formal proof is a mathematical object just as a formula/statement/expression is, and one can think of the proving process as "solving" for a proof.

The relationship between the two approaches is usually expressed as questioning the validity of the proof method. You want to ensure two things:

  • soundness (aka consistency, proofs only exist for true statements)
  • completeness (a proof exists for any true statement)
where "true statement" is taken as "something that evaluates to true".

Due to its mechanical nature, formal logic is perfectly suited for being interpreted mechanically. This is in itself a vast field of research.

A brief explanation of

Modal logic

Very basic formal logics, the kind that are the most thoroughly studied, particularly by nonspecialists like me, deal with a single state of the universe. A proposition is true, or it is false. There is no sense of what it has been, or what it could be, or how it might get to be that way. A natural way to understand modal logic is that it allows us to work with alternate universes.

Consider the passage of time. We might want to say that p will be true at some time in the future. This would be expressed in temporal logic as Fp.

The F operator introduces another time relative to the current one, and all propositions in the context of that operator are making claims about a point in the future. There is also an operator G. Gp means p will always be true in the future, though note that that doesn't mean p is currently true. That is, it is not the case that $\text{F}p \rightarrow p$.

What is perhaps a bit more interesting is to say "$p$ will be true at some time in the future, and thereafter it will always be true". This would be expressed as $\text{F}(p \wedge \text{G}p)$.

You can see that F is interpreted as "at some time in the Future ...", while G is "it is always Going to be ...".
